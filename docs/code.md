# Code Notes

## Language and frameworks

- Backend: C#, ASP.NET Core (.NET 10), EF Core, MySQL (Pomelo provider).
- Frontend: C#, .NET MAUI, Mapsui, CommunityToolkit.Maui, Syncfusion, LocalNotification.

## Generated code

- `frontend/mobile/NiceCleanApp/Services/ApiClient.cs` is generated by NSwag at build time.
- The Swagger source is downloaded from:
  - `https://nicecleanrest.azurewebsites.net/swagger/v1/swagger.json`
- Generation step is defined in `NiceCleanApp.csproj`.
- Do not hand-edit generated regions; regenerate by building the MAUI project.

## Dependency injection

- Backend uses ASP.NET Core DI to register repositories and DbContext.
- Mobile app uses MAUI DI to register pages and services.

## Repository strategy

- In-memory repositories are used by default:
  - `PinRepository`, `UserRepository`, `EventRepository`, `ReportRepository`, `PinVoteRepository`.
- EF Core repositories are available:
  - `PinRepositoryDB`, `UserRepositoryDB`, `EventRepositoryDB`, `ReportRepositoryDB`, `PinVoteRepositoryDB`.
- Switching to DB repositories requires updating `Program.cs` service registrations.

## Important implementation notes

- `EventRepository` (in-memory) contains a `_userRepository` field but it is not set.
  - `GetEventNicknames` uses this field; calling it without wiring a user repository will throw.
  - This method is not called by the current controllers, but it is a correctness risk.
- `PinValidationPopup` uses `.Result` on async API calls.
  - This blocks the UI thread and can deadlock on some platforms. It is currently used for button visibility.
- Event report submission is not wired to the API (see `EventDetailPopup`).

## Security and privacy considerations

- Login uses plain email/password with no hashing or token-based session.
- CORS is wide open by default.
- Sensitive data (passwords) are stored in the mobile app via secure storage, but the backend still stores and returns raw passwords.
